# Output Overview

This page describes the output directory structure generated by the HEST-Radiomics pipeline.

The pipeline organizes all intermediate and final outputs under the dataset-specific directory.

Example:

```text
data/
└── hest/
    └── IDC/
        ├── .cache/
        ├── genes/
        ├── metadata/
        ├── patches/
        ├── patches_vis/
        ├── radiomics/
        ├── radiomics_stats/
        ├── segment/
        ├── segment_vis/
        ├── spatial_plots/
        ├── st/
        └── thumbnails/
````

Each directory serves a different role in the spatial transcriptomics and radiomics workflow.

---

## Directory Descriptions

#### .cache/

Temporary cache files used during preprocessing and feature extraction.

Typical contents may include:

* intermediate preprocessing artifacts
* cached metadata
* multiprocessing cache files
* temporary feature representations

This directory can usually be regenerated safely.

---

#### genes/

Gene-related outputs and processed transcriptomics information.

Typical contents may include:

* highly variable gene (HVG) lists
* selected gene panels
* normalized gene metadata
* gene filtering results

Example:

```text
genes/
├── var_250genes.json
└── ...
```

---

#### metadata/

Dataset-level metadata files.

Typical contents may include:

* sample metadata
* patient information
* dataset annotations
* experimental configuration files

---

#### patches/

HDF5 patch datasets extracted from Whole Slide Images (WSIs).

Each `.h5` file typically contains:

* patch images
* patch coordinates
* spatial transcriptomics barcodes

Example:

```text
patches/
├── NCBI681.h5
├── NCBI682.h5
└── ...
```

These files are commonly used as the primary image source for downstream analysis.

---

#### patches_vis/

Visualization outputs for extracted patches.

Typical outputs include:

* patch grids
* sampled patch previews
* quality control visualizations

These files are intended for human inspection and debugging.

---

#### radiomics/

Extracted radiomics feature files.

Typical outputs include:

* patch-level radiomics features
* cell-aware radiomics features
* PyRadiomics-based handcrafted descriptors

Depending on the pipeline configuration, outputs may be stored in:

* `.parquet`
* `.csv`
* `.h5ad`

Example:

```text
radiomics/
├── NCBI681.h5ad
├── NCBI682.h5ad
└── ...
```

---

#### radiomics_stats/

Statistical summaries and diagnostic outputs for radiomics features.

Typical outputs include:

* feature distributions
* normalization statistics
* clipping statistics
* feature variance analysis
* summary plots

This directory is useful for feature quality assessment and debugging.

---

#### segment/

Cell segmentation outputs stored in HDF5 format.

Each segmentation file contains:

* polygon-based cell boundaries
* cell type annotations
* patch-to-cell mappings
* segmentation metadata

Example:

```text
segment/
├── NCBI681.h5
├── NCBI682.h5
└── ...
```

See also:

* `outputs/segment.md`

---

#### segment_vis/

Visualization outputs for segmentation results.

Typical outputs include:

* segmentation overlays
* polygon rendering
* cell boundary visualizations
* class-colored segmentation maps

These outputs are useful for validating segmentation quality.

---

#### spatial_plots/

Spatial transcriptomics visualization outputs.

Typical plots may include:

* gene expression heatmaps
* spatial feature maps
* cluster visualizations
* tissue overlays

These visualizations help interpret spatial biological patterns.

---

#### st/

Spatial transcriptomics datasets.

Typically stored in H5AD format.

Example:

```text
st/
├── NCBI681.h5ad
├── NCBI682.h5ad
└── ...
```

These files commonly contain:

* gene expression matrices
* spot coordinates
* transcriptomics metadata
* spatial annotations

---

#### thumbnails/

Low-resolution thumbnail images of WSIs.

Used for:

* quick preview
* visualization
* coordinate alignment
* debugging

These images provide a lightweight representation of the original slide.

---

## Notes

* Most outputs are generated automatically by the pipeline.
* Intermediate directories may vary depending on the extraction configuration.
* Some visualization directories are optional.
* Large datasets may require substantial storage space.
* HDF5-based storage is used extensively for scalability and efficient I/O.
